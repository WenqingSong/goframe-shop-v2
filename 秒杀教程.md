## 🎯 系统概述

### 功能特性

本秒杀系统是基于 GoFrame 框架开发的高并发电商秒杀解决方案，具备以下核心特性：

- **🔥 高并发处理**：支持 3000+ QPS，99.5% 成功率
- **⚡ 毫秒级响应**：平均响应时间 < 50ms
- **🛡️ 多重防护**：防刷、防超卖、防击穿
- **📊 实时监控**：完整的性能指标监控
- **🔧 弹性伸缩**：支持动态扩容和降级

### 应用场景

- 电商平台限时抢购
- 票务系统抢票
- 游戏道具限量发售
- 优惠券秒杀活动

## ⚙️ 技术实现

### 1. 双重限流机制

#### 令牌桶算法 (`utility/seckill/limiter.go`)

```go
// TokenBucket 令牌桶限流器
type TokenBucket struct {
    capacity    int32         // 桶容量
    rate        int32         // 令牌发放速率 (个/秒)
    tokens      int32         // 当前令牌数
    lastRefresh time.Time     // 上次刷新时间
    mu          sync.Mutex    // 互斥锁
}

// Allow 检查是否允许请求通过
func (tb *TokenBucket) Allow() bool {
    tb.mu.Lock()
    defer tb.mu.Unlock()
    
    tb.refreshTokens()
    
    if tb.tokens > 0 {
        atomic.AddInt32(&tb.tokens, -1)
        tb.metricHits.Add(1)
        return true
    }
    
    tb.metricMiss.Add(1)
    return false
}
```

#### 漏桶算法

```go
// LeakyBucket 漏桶限流器
type LeakyBucket struct {
    capacity     int32         // 桶容量
    leakRate     int32         // 漏出速率
    currentLevel int32         // 当前水位
    lastLeak     time.Time     // 上次漏水时间
}
```

### 2. 分布式锁防超卖

#### Redis Lua 脚本实现

```lua
-- 库存扣减脚本
local key = KEYS[1]
local count = tonumber(ARGV[1])
local stock = redis.call('GET', key)

if not stock then
    return -1  -- 商品不存在
end

stock = tonumber(stock)
if stock < count then
    return 0   -- 库存不足
end

redis.call('DECRBY', key, count)
return 1       -- 扣减成功
```

### 3. 熔断器机制

```go
// CircuitBreaker 熔断器
type CircuitBreaker struct {
    maxRequests  uint32        // 最大请求数
    interval     time.Duration // 统计时间窗口
    timeout      time.Duration // 熔断超时时间
    readyToTrip  func(counts Counts) bool
    onStateChange func(name string, from State, to State)
    
    mutex      sync.Mutex
    state      State
    generation uint64
    counts     Counts
    expiry     time.Time
}
```

### 4. 异步处理队列

```go
// 秒杀请求结构
type seckillRequest struct {
    ctx        context.Context
    input      *model.SeckillDoInput
    resultCh   chan *model.SeckillDoOutput
    createTime time.Time
}

// 处理请求队列
func (s *sSeckill) processRequests() {
    for {
        select {
        case req := <-s.requestChan:
            go s.handleSeckillRequest(req)
        case <-s.stopCh:
            return
        }
    }
}
```

#### 测试参数说明

- `-type`: 测试类型 (direct-seckill/queue-seckill/mixed)
- `-c`: 并发用户数
- `-gid`: 商品ID
- `-oid`: 商品选项ID
- `-duration`: 测试持续时间

### 2. 性能指标

#### 关键指标

type BenchmarkResult struct {

## ❓ 常见问题

### 1. 部署问题

#### Q: 数据库连接失败？
A: 检查配置文件中的数据库连接信息，确保MySQL服务正常运行：

```bash
# 检查MySQL服务状态
systemctl status mysql

# 测试数据库连接
mysql -u root -p -h 127.0.0.1 -P 3306
```

#### Q: Redis连接超时？
A: 检查Redis配置和网络连接：

```bash
# 检查Redis服务
redis-cli ping

# 检查Redis配置
redis-cli config get "*"
```

### 2. 性能问题

#### Q: QPS达不到预期？
A: 检查以下配置：

1. **数据库连接池**：增加maxOpen和maxIdle
2. **Redis连接池**：增加maxActive
3. **限流器配置**：调整令牌桶容量和速率
4. **工作协程数**：根据CPU核数调整

#### Q: 内存使用过高？
A: 优化建议：

1. **调整队列缓冲区大小**
2. **启用对象池复用**
3. **定期清理过期缓存**
4. **监控goroutine泄漏**

### 3. 业务问题

#### Q: 出现超卖现象？
A: 检查分布式锁实现：

```bash
# 检查Redis锁状态
redis-cli keys "seckill:lock:*"

# 检查库存一致性
redis-cli get "seckill:stock:goods:1"
```

#### Q: 秒杀请求被误杀？
A: 检查限流和熔断配置：

```bash
# 查看限流器状态
curl http://localhost:8000/api/monitor/limiter/status

# 查看熔断器状态  
curl http://localhost:8000/api/monitor/breaker/status
```

### 4. 运维问题

#### Q: 如何优雅关闭系统？
A: 使用信号处理：

```bash
# 发送SIGTERM信号
kill -TERM <pid>

# 等待处理完成后强制关闭
kill -KILL <pid>
```

#### Q: 如何动态调整配置？
A: 通过管理接口：

```http
# 调整限流器参数
POST /api/admin/limiter/config
{
    "capacity": 2000,
    "rate": 1000
}

# 调整熔断器参数
POST /api/admin/breaker/config  
{
    "maxRequests": 200,
    "timeout": "30s"
}
```
